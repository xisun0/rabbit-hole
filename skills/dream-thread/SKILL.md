# Dream Thread

Dream Thread is a recombinatory ideation skill inspired by dream-like cognition.

It is designed for moments when the user is stuck inside familiar assumptions, when ordinary brainstorming becomes repetitive, or when a project needs new questions rather than quick conclusions.

This skill does not use dream language to decorate an answer.
It uses dream-like recombination to generate candidate connections, then filters which ones survive logical review.

## When to use

Use Dream Thread when:

- the user already has material, but ordinary analysis only repeats known frames
- the user needs fresh questions, hypotheses, or conceptual directions
- there is an unresolved tension, unexplained gap, or awkward boundary in the material
- brainstorming is converging too quickly
- the task benefits from strange but testable new connections

Do not use Dream Thread when:

- the user only needs factual lookup
- the task requires a standard, direct workflow
- the user already has a clear conclusion and only needs rewriting
- the task does not allow exploratory deviation

## Core idea

Dream Thread follows a staged process:

1. prepare the dream
2. suspend ordinary logic
3. recombine concepts through structured operations
4. reopen logic and filter what survives
5. translate surviving links into questions or hypotheses
6. validate before turning anything into recommendations

The purpose is not to reach a conclusion immediately.

The purpose is to produce:
- candidate questions
- candidate hypotheses
- conceptual tensions worth pursuing
- possible new directions for research, design, or writing

## Two phases

### Phase 1: Prepare the dream
In this phase, reading files and using available materials is allowed.

Complete three tasks:

1. **Extract concept nodes**
   Identify 5–8 independent concept nodes from the user's material.

   A concept node can be:
   - an object
   - a mechanism
   - a constraint
   - an actor
   - a repeated action
   - a time structure
   - an unresolved tension

   Prefer raw material over conclusions.
   Do not extract finished arguments if smaller, recombinable parts are available.

2. **Inject foreign concepts**
   Introduce 1–2 distant concepts from unrelated domains.

   Good source domains may include:
   - ecology
   - weather systems
   - logistics networks
   - geology
   - ritual structures
   - urban infrastructure
   - material fatigue
   - animal behavior

   The goal is not metaphor for its own sake.
   The goal is to force connections that would not normally appear.

3. **State the current uncertainty**
   Write down 1–2 open questions.
   Dream Thread runs on uncertainty, not on conclusions.

### Phase 2: Run the dream
In this phase, do not read new files and do not search for new information.

Use only the nodes, foreign concepts, and open questions already present in context.

Each round should apply one recombination operation with logic temporarily loosened.

Allow:
- strangeness
- contradiction
- incomplete structures
- unstable combinations

Do not explain the dream while still inside it.

## Recombination operations

In each round, choose one operation and actually execute it.

### Transplant
Install the mechanism of a foreign concept into a local concept.

### Inversion
Reverse a familiar causal direction.

### Scale mismatch
Force a concept to operate at the wrong time or spatial scale.

### Role exchange
Swap the subject and object inside a relation.

### Boundary magnification
Stay at the seam between two concepts and inspect what appears there.

### Absence experiment
Remove one crucial concept and ask what takes its place.

### Fusion
Collapse two concepts into one hybrid form and inspect its new structure.

## Dream rounds

Run 3 to 5 rounds depending on complexity.

For each round:

1. name the operation
2. run the recombination
3. leave behind one suspended image, tension, or unresolved question

Each round must create a meaningful shift.
Do not simply describe the same dream in different words.

Soft cues may be used between rounds, but they should not push toward a conclusion.

Examples of soft cues:
- What moved first?
- What no longer fits here?
- Which boundary has become visible?
- What seems to be missing?
- What changed scale?
- What now feels reversed?
- What should not belong here, but does?

## Filtering

After the dream rounds, reopen logical review.

For each suspended image or unresolved question, classify it as:

- **Survives**: logically meaningful and worth pursuing
- **Partially survives**: contains an interesting core but needs revision
- **Dissolves**: noise, discard

Do not force meaning onto weak material.
Failure is normal. Most dream links should dissolve.

## Return to reality

Only translate items that survive or partially survive.

Convert them into one or more of the following:

- a research question
- a hypothesis
- a conceptual reframing
- a design direction
- a writing angle
- a structural problem worth investigating

Do not produce recommendations yet unless validation has already happened.

## Validation

After translation, check whether each question or hypothesis corresponds to a real gap in the source material.

Validation questions:
- Does this point back to actual concept nodes from the material?
- Does it reveal a real tension, mechanism, or missing explanation?
- Is it merely dream noise, or does it change how the problem should be understood?
- If true, would it alter the next step in research, writing, or design?

Only after validation should the output become an action recommendation.

## Guardrails

- Keep the dream grounded in the user's actual material.
- Do not gather new information during the dream phase.
- Do not confuse novelty with validity.
- Do not smuggle conclusions into dream language.
- Prefer questions and hypotheses over finished answers.
- Let weak links dissolve.
- The dream phase generates material; the filtering phase judges it.
