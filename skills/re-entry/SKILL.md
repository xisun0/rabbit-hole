# Re-entry

Re-entry is the waking-phase translation skill in Rabbit Hole.

It takes the surviving material produced by dream-like recombination and brings it back into practical language.

This skill does not generate new dream content.
It does not extend surreal logic.
It does not treat every strange image as meaningful.

Its job is to:

- identify which dream elements matter
- map them back to the user's actual material
- translate them into clear questions, hypotheses, or conceptual directions
- separate signal from decorative residue
- prepare outputs for validation

## When to use

Use Re-entry when:

- a prior recombination step has already produced dream material
- there are suspended images, tensions, or strange pairings that need translation
- the user needs practical meaning, not more dream expansion
- the project is ready to move from exploration into interpretation
- candidate connections need to be reframed as usable next-step thinking

Do not use Re-entry when:

- no recombination has happened yet
- the material is still too raw and unformed to interpret
- the task requires more divergence rather than consolidation
- the user only wants poetic continuation
- there is no source problem to map back to

## Core idea

Dream material is not useful by itself.

Its value depends on whether it can be translated back into the structure of the original problem.

Re-entry works by asking:

- What does this image correspond to in the source material?
- What relation or tension does it reveal?
- Is it pointing to a mechanism, a missing variable, a misframed question, or a structural gap?
- Does it survive as a serious line of thought, or only as atmosphere?

The goal is not to preserve dream language.
The goal is to preserve whatever insight the dream material may contain.

## Inputs

Re-entry expects some or all of the following:

- the original problem or task
- concept nodes from the preparation phase
- foreign concepts introduced during recombination
- suspended images or unresolved questions from dream rounds
- survival labels such as:
  - survives
  - partially survives
  - dissolves

If the source problem is unclear, restate it first before translating anything.

## Translation process

### Step 1: Restate the waking problem
Begin by stating the real problem in clear waking language.

This should be short and concrete.

Do not repeat the dream.
Do not add interpretation yet.

### Step 2: Gather only viable dream material
Only work with dream elements that are marked:

- survives
- partially survives

Ignore dissolved material unless it helps clarify why something failed.

### Step 3: Map dream material back to source structure
For each surviving item, identify what it corresponds to.

A dream element may map back to:

- a concept node
- a relation between nodes
- a hidden assumption
- a missing actor
- a broken causal chain
- a scale mismatch
- a boundary problem
- an unexamined dependency
- a repeated tension

Do not stop at symbolism alone.
Move from image to structure.

### Step 4: Name the type of insight
For each translated item, decide what kind of waking output it is.

Possible types include:

- research question
- hypothesis
- conceptual reframing
- mechanism candidate
- design direction
- writing angle
- decision tension
- structural diagnostic

### Step 5: Rewrite in waking language
Rewrite each item so it becomes explicit and discussable.

Good waking language is:

- clear
- compact
- structurally meaningful
- testable when possible

Avoid:
- mystical phrasing
- dream residue that does not help reasoning
- inflated abstraction
- recommendations that skip validation

### Step 6: Preserve uncertainty
Do not over-stabilize weak material.

If something is interesting but not yet strong, present it as:

- a possible hypothesis
- a candidate reframing
- a tension worth checking
- a conditional interpretation

Re-entry should clarify without pretending certainty.

## What Re-entry should produce

Re-entry may produce one or more of the following:

- a more precise research question
- a hypothesis worth checking
- a reframed version of the original problem
- a newly visible mechanism
- a boundary condition that had been ignored
- a more useful writing frame
- a structured next-step inquiry

It should usually not produce:

- final conclusions
- polished recommendations without validation
- purely aesthetic summaries of the dream
- new dream imagery

## Translation heuristics

Use these heuristics when translating dream material.

### From image to mechanism
Ask:
- What is this image doing?
- What relation does it imply?
- What process does it resemble?

Translate actions before symbols whenever possible.

### From pairing to tension
If two unlike things were fused or forced together, ask:
- What incompatibility or hidden dependency does this reveal?
- Why did these two concepts need each other?

### From distortion to misframing
If scale, role, or causality was distorted, ask:
- Was the original problem framed at the wrong scale?
- Was the wrong actor treated as central?
- Was a consequence mistaken for a cause?

### From absence to hidden support
If something was removed, ask:
- What normally fills this function?
- What invisible support structure became visible only after removal?

### From residue to discard
If a dream element cannot be mapped back to any meaningful structure, let it go.

Not every surviving image deserves interpretation.

## Guardrails

- Do not generate new dream content.
- Do not continue surreal expansion.
- Do not treat every strange image as signal.
- Do not jump directly from dream material to recommendation.
- Keep uncertainty visible where uncertainty remains.
- Translate into structure, not just into prettier wording.
- Prefer questions and hypotheses over premature closure.
- Stay accountable to the user's original material and task.

## Relationship to Dream Thread

Dream Thread generates candidate links under loosened logic.

Re-entry reopens waking logic and asks which of those links can be meaningfully translated back into the original problem.

Dream Thread expands.
Re-entry stabilizes.

Dream Thread suspends judgment.
Re-entry restores disciplined interpretation.
